J.U.C.locks(1)--AQS

本文最后更新于：2022年8月7日 晚上

简介

在locks包下有一个类是AbstractQueuedSynchronizer，其简写就是AQS，AQS是一个用来构建锁和同步器的框架，使用AQS能简单且高效地构造出应用广泛的大量的同步器。

AQS核心思想是，如果被请求的共享资源空闲，则将当前请求资源的线程设置为有效的工作线程，并且将共享资源设置为锁定状态；如果被请求的共享资源被占用，那么就需要一套线程阻塞等待以及被唤醒时锁分配的机制，这个机制AQS是用CLH队列锁实现的，即将暂时获取不到锁的线程加入到队列中。

CLH(Craig,Landin,and Hagersten)队列是一个虚拟的双向队列(虚拟的双向队列即不存在队列实例，仅存在结点之间的关联关系)。AQS是将每条请求共享资源的线程封装成一个CLH锁队列的一个结点(Node)来实现锁的分配。Node其实就是AQS里面的一个内部类

在AQS中，资源存在两种：独占资源和共享资源。最简单的例子就是，如果一个资源只能被读取而不能被修改，那么这个资源就可以作为共享资源。之前也提到AQS相当于是一个框架，自定义同步器在实现时只需要实现共享资源 state 的获取与释放方式即可（其实也就是锁的最基本特性），至于具体线程等待队列的维护(如获取资源失败入队/唤醒出队等)，AQS已经在上层已经帮我们实现好了。

Condition

在Java API里面对Condition接口的解释如下

Condition factors out the Object monitor methods (wait, notify and notifyAll) into distinct objects to give the effect of having multiple wait-sets per object, by combining them with the use of arbitrary Lock implementations. Where a Lock replaces the use of synchronized methods and statements, a Condition replaces the use of the Object monitor methods.

大致意思是，之前我们使用synchronized关键字实现同步方法和语句，如果一个线程需要等待或者唤醒其他线程，就需要用到锁对象中的wait/notify方法。现在Condition代替了锁对象的方法，Lock代替了synchronized的使用。

所以在Condition中定义的方法也很简单，通过查看相关API一目了然。
